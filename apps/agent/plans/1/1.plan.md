# 1. 墒情 Agent 当前实施方案

## 目标

当前墒情 Agent 只做一件事：围绕真实墒情数据给出受控、可核验、可追溯的回答。

- 事实来源只认 `fact_soil_moisture`
- 查询时间统一使用 `create_time`
- 原始 Excel 字段名是唯一数据契约
- LLM 是意图理解与工具选择的决策者
- 代码负责参数验证、SQL 执行、事实核查三条安全边界

## 当前数据契约

### 事实表原则

- 墒情事实唯一表：`infra/mysql/init/001_init_tables.sql` 中的 `fact_soil_moisture`
- 原始字段直接保留：`id`、`sn`、`gatewayid`、`sensorid`、`unitid`、`city`、`county`、`time`、`create_time`、各层水分温度与字段状态、`lat`、`lon`
- 管理追溯字段只保留：`source_file`、`source_sheet`、`source_row`
- 查询执行、排序、最新业务时间、时间过滤全部使用 `create_time`

### 配套表

- `soil_import_job` / `soil_import_job_diff`：导入预览与差异快照
- `region_alias`：`city` / `county` 两级别名映射
- `metric_rule`：规则配置
- `warning_template`：模板配置
- `agent_query_log`：查询审计

## 固定主流程（5 节点）

```text
用户问题
  -> InputGuard         （安全门：拦截非业务/越界输入）
  -> AgentLoop          （LLM + Function Calling 核心循环）
  -> DataFactCheck      （事实核查）
  -> AnswerVerify       （回答合规校验）
  -> FallbackGuard      （统一兜底）
```

## 节点职责

### InputGuard

- 识别问候、乱输、越界、纯结束语
- 非业务输入不进查库链路
- 纯结束语命中时清空上下文

### AgentLoop

- 加载会话消息历史（Redis）
- 构造 system_prompt + history + user_message 发给 LLM
- LLM 返回 tool_call → ToolExecutorService 验证 + 执行 → 追加结果 → 循环
- LLM 返回 text → 作为最终回答，保存本轮消息历史
- 最多 5 次工具调用，超限进入 FallbackGuard

### DataFactCheck

- 核对回答中的数字、时间、地区、设备号
- 防止空回答、错对象、半成品模板和内部实现术语外泄

### AnswerVerify

- 最终回答合规校验
- 通过则 verified_end，失败则 fallback_guard

### FallbackGuard

- 统一兜底节点，返回安全提示

## 工具集契约（7 个工具）

工具定义见 `app/llm/tools/soil_tools.py`。

| 工具名 | 用途 |
|--------|------|
| `get_soil_overview` | 整体概况 |
| `get_soil_ranking` | 排名对比（需 aggregation 参数） |
| `get_soil_detail` | 区域/设备详情 |
| `get_soil_anomaly` | 异常查询 |
| `get_warning_data` | 预警数据 |
| `get_advice_context` | 建议背景数据 |
| `diagnose_empty_result` | 空结果诊断（需 scenario 参数） |

所有工具必须传 `start_time` / `end_time`（格式 `YYYY-MM-DD HH:MM:SS`）。

## 工具参数安全边界

由 `ToolExecutorService` 强制执行：

- `top_n` 最大 20
- 设备级排名时间跨度最大 365 天
- 异常查询时间跨度最大 180 天
- 工具名必须在白名单内（7 个）

验证失败时将 error 消息返回给 LLM 让其调整，不终止循环。

## 多轮对话契约

Redis 存储真实对话消息，LLM 从历史中天然解析多轮上下文。

- Redis key: `session_msg:{session_id}`
- 存储格式：OpenAI messages 格式（user / assistant / tool）
- 最大 20 条消息，TTL 7200 秒
- 实现：`app/repositories/session_context_repository.py`
  - `load_history(session_id)` → `list[dict]`
  - `save_message_turn(session_id, turn_id, *, user_message, assistant_message, tool_calls, tool_results)`
  - `clear_context(session_id)`
