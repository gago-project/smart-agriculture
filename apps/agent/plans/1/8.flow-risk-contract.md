# 8. 墒情 Agent Flow 安全契约（LLM + Function Calling 版）

> 本文档描述 LLM + Function Calling 架构下的工程风险约束与降级策略。

---

## 1. 五类风险契约

### 1.1 路由安全契约（不变）

- 节点声明 `allowed_next_actions` 与路由表必须完全一致
- Flow 启动时执行节点/路由/动作完整性自检（`RouteRegistry.validate()`）
- `fallback` 不得直接等于终止态，必须走 `FallbackGuard`

### 1.2 状态安全契约（不变）

- 每个节点有 `allowed_patch_fields` 白名单
- `merge_state_patch()` 发现越权字段时直接报错
- 当前核心写权限：
  - `answer_bundle.final_answer`：`AgentLoopNode`
  - `query_result.records`：`AgentLoopNode`
  - `answer_type`：`AgentLoopNode` / `DataFactCheckNode`
  - `input_type`：`InputGuardNode`
  - `conversation_closed`：`InputGuardNode`

### 1.3 循环安全契约（双层保护）

- `AgentLoopService` 内部：最多 5 次工具调用（`MAX_TOOL_ITERATIONS=5`），超限强制终止并返回兜底消息
- `FlowRunner` 外部：`max_steps=32` / `max_retry_count=2` / `max_node_visits_per_node=4`
- 任一超限进入 `FallbackGuard`

### 1.4 上下文安全契约（重设计）

旧：槽位白名单继承（`entity_context` + `query_frame` + `resolved_window`）

新：完整消息历史（user / assistant / tool），LLM 天然解析上下文

- Redis key: `session_msg:{session_id}`
- 最大 20 条消息，TTL 7200 秒
- Redis 不可用时退回单轮模式，不拖垮主查询

### 1.5 知识边界安全契约

- LLM 只能基于 `ToolExecutorService` 返回的数据回答
- LLM 不得在回答中引用外部知识作为数字事实
- `DataFactCheckNode` 负责比对回答内容与 `query_result`

---

## 2. 新增风险：LLM 幻觉工具参数

### 风险描述

LLM 可能生成不合法的工具参数（无效地区名、超范围时间跨度、超大 top_n）。

### 硬约束

- `ToolExecutorService` 是唯一执行层，所有参数必须经过验证
- 验证失败时将 error 消息返回给 LLM（不终止循环），让 LLM 调整
- 连续验证失败超过 `MAX_TOOL_ITERATIONS` 次时，终止并降级

---

## 3. 新增风险：LLM 不可用

### 风险描述

Qwen API 不可用时，`AgentLoopService` 无法继续，整个业务路径失效。

### 硬约束（与旧架构不同）

- Qwen 不可用 = 必须明确报错，不能假装正常（旧架构可退回正则解析）
- 返回明确的服务不可用提示，前端应引导用户等待
- `AgentLoopService` 检测到 `qwen_client.available() == False` 时，立即返回 `is_fallback=True`

---

## 4. 降级优先级

1. **继续执行但降能力**：Redis 不可用 → 无历史，退回单轮模式
2. **工具验证失败**：返回 error 给 LLM，让 LLM 调整参数后重试
3. **工具调用超上限**：强制终止，返回兜底消息
4. **DataFactCheck / AnswerVerify 失败**：`FallbackGuard` 保守兜底
5. **LLM 不可用**：立即报错，不降级继续
6. **DB 不可用**：HTTP 503，不伪造数据

---

- [ ] `SessionContextRepository` Redis 失败时返回空列表（单轮模式），已有测试覆盖
