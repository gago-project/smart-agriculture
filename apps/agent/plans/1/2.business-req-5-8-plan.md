## 一、数据可行性分析

### 需求 #5：全省设备城市分布

**问法示例：** "土壤墒情仪分布在哪里？" / "江苏省各地市有多少台墒情仪？"

| 条件 | 结论 |
|------|------|
| 数据源 | `subject_device_record`，字段 `city`、`type='土壤墒情仪'`，按 city GROUP BY COUNT |
| 数据就绪 | 表存在（`005_add_subject_device_record.sql`），需 Excel 导入后才有行数据 |
| 城市归一化 | Excel 中 `city` 字段可能是"南京"（无"市"后缀），需在 repo 层加后缀补全逻辑 |
| 固定顺序 | 江苏 13 市固定枚举在代码里：南京市、无锡市、常州市、苏州市、镇江市、南通市、扬州市、泰州市、徐州市、连云港市、淮安市、盐城市、宿迁市 |
| **结论** | **数据结构支持，表建好后可实现** |

### 需求 #6：指定城市设备县区分布

**问法示例：** "南通市土壤墒情仪分布情况" / "南通的墒情设备各县区有多少"

| 条件 | 结论 |
|------|------|
| 数据源 | `subject_device_record`，`WHERE city = 'X' AND type = '土壤墒情仪'`，GROUP BY `county` |
| 参数来源 | 用户消息中提取 city 实体，由 `_entities(city=...)` 传入 |
| 数据就绪 | 同 #5，依赖 Excel 导入 |
| **结论** | **数据结构支持，需提取城市实体** |

### 需求 #7：预警规则说明

**问法示例：** "土壤墒情的预警规则是什么？" / "什么情况下会触发重旱预警？"

| 条件 | 结论 |
|------|------|
| 数据源 | `metric_rule` 表，`rule_code = 'soil_warning_v1'`；`warning_rule_row()` 方法已存在（`soil_repository.py` line 524） |
| 内容 | 涝渍：SRWC ≥ 150%，重旱：SRWC < 50%，设备故障：water20cm=0 AND t20cm=0 |
| 静态性 | 规则几乎不变；从 DB 读 + 格式化，失败时硬编码兜底 |
| **结论** | **数据结构完全支持，`warning_rule_row()` 已有** |

### 需求 #8：预警记录查询

**问法示例：** "最近7天南通市哪些点位有预警？" / "上周有多少条重旱预警记录？"

| 条件 | 结论 |
|------|------|
| 数据源 | `fact_soil_moisture`：`water20cm < 50`（重旱）或 `water20cm >= 150`（涝渍）或 `water20cm=0 AND t20cm=0`（故障） |
| 时间过滤 | `create_time BETWEEN start AND end`，已有 `filter_records` 基础结构 |
| 地区过滤 | 同现有查询，`city`/`county`/`sn` 参数 |
| SQL 阈值 | 当前硬编码在 SQL WHERE 里（P0 升级时改为从 `metric_rule` 表读取） |
| **结论** | **数据结构完全支持，`fact_soil_moisture` 有足够字段** |

> **重要**：`fact_soil_moisture` 没有 `soil_status` 列，soil_status 由 Python 实时计算。
> 预警查询必须在 SQL WHERE 中直接写阈值条件，不能依赖 Python 计算后再过滤。

```python
@staticmethod
def build_soil_device_city_distribution_audit_sql() -> str:
    return (
        "SELECT city, COUNT(*) AS device_count\n"
        "FROM subject_device_record\n"
        "WHERE type = '土壤墒情仪'\n"
        "GROUP BY city\n"
        "ORDER BY FIELD(city, '南京市','无锡市','常州市','苏州市','镇江市',"
        "'南通市','扬州市','泰州市','徐州市','连云港市','淮安市','盐城市','宿迁市')"

```

**方法 B：同步执行**

```python
def soil_device_city_distribution(self) -> list[dict] | None:
    """
    返回列表：[{"city": "南通市", "device_count": 12}, ...]
    顺序按江苏 13 市固定排列；有数据的市才出现。
    表不存在时返回 None。
    """
    _JIANGSU_CITY_ORDER = [
        "南京市", "无锡市", "常州市", "苏州市", "镇江市",
        "南通市", "扬州市", "泰州市", "徐州市", "连云港市",
        "淮安市", "盐城市", "宿迁市",
    ]
    connection = self._connect()
    try:
        with connection.cursor() as cursor:
            cursor.execute(
                "SELECT city, COUNT(*) AS device_count "
                "FROM subject_device_record "
                "WHERE type = %s "
                "GROUP BY city",
                ("土壤墒情仪",),
            )
            rows = cursor.fetchall()
        # 归一化城市名（补"市"后缀），然后按固定顺序排列
        normalized: dict[str, int] = {}
        for row in rows:
            city = row.get("city") or ""
            if city and not city.endswith("市") and not city.endswith("港市"):
                city = city + "市"
            normalized[city] = int(row.get("device_count") or 0)
        result = [
            {"city": c, "device_count": normalized.get(c, 0)}
            for c in _JIANGSU_CITY_ORDER
            if c in normalized
        ]
        return result if result else None
    except Exception as exc:
        message = str(exc)
        if "subject_device_record" in message and ("1146" in message or "doesn't exist" in message):
            return None
        raise DatabaseQueryError(f"MySQL 查询设备城市分布失败：{exc}") from exc
    finally:
        connection.close()
```

**方法 C：异步包装**

```python
async def soil_device_city_distribution_async(self) -> list[dict] | None:
    return await asyncio.to_thread(self.soil_device_city_distribution)
```

```python
def soil_device_county_distribution(self, city: str) -> list[dict] | None:
    """
    返回列表：[{"county": "如东县", "device_count": 5}, ...]
    city 参数自动补"市"后缀；返回 None 表示表不存在或无数据。
    """
    if city and not city.endswith("市") and not city.endswith("港市"):
        city = city + "市"

    connection = self._connect()
    try:
        with connection.cursor() as cursor:
            cursor.execute(
                "SELECT county, COUNT(*) AS device_count "
                "FROM subject_device_record "
                "WHERE type = %s AND city = %s "
                "GROUP BY county "
                "ORDER BY device_count DESC",
                ("土壤墒情仪", city),
            )
            rows = cursor.fetchall()
        if not rows:
            return None
        return [
            {
                "county": row.get("county") or "（未知）",
                "device_count": int(row.get("device_count") or 0),
            }
            for row in rows
        ]

        message = str(exc)
        if "subject_device_record" in message and ("1146" in message or "doesn't exist" in message):
            return None
        raise DatabaseQueryError(f"MySQL 查询设备县区分布失败：{exc}") from exc
```

```python
def query_warning_records(
    self,
    *,
    city: str | None = None,
    county: str | None = None,
    sn: str | None = None,
    start_time: str | None = None,
    end_time: str | None = None,
    warning_type: str | None = None,
    limit: int = 50,
) -> list[dict]:
    """
    返回满足预警条件的记录列表。
    每条记录含 sn, city, county, create_time, water20cm, water40cm, warning_level。
    warning_level 由 SQL CASE 表达式直接计算（heavy_drought / waterlogging / device_fault）。
    阈值硬编码：重旱 <50，涝渍 >=150，故障 water20cm=0 AND t20cm=0。
    """
    clauses: list[str] = []
    params: list = []

    for col, op, val in [
        ("city", "=", city),
        ("county", "=", county),
        ("sn", "=", sn),
        ("create_time", ">=", start_time),
        ("create_time", "<=", end_time),
    ]:
        if val is not None:
            clauses.append(f"{col} {op} %s")
            params.append(val)

    if warning_type == "heavy_drought":
        clauses.append("water20cm < 50")
    elif warning_type == "waterlogging":
        clauses.append("water20cm >= 150")
    elif warning_type == "device_fault":
        clauses.append("(water20cm = 0 AND t20cm = 0)")
    else:
        clauses.append("(water20cm < 50 OR water20cm >= 150 OR (water20cm = 0 AND t20cm = 0))")

    where = f"WHERE {' AND '.join(clauses)}" if clauses else ""
    sql = f"""
    SELECT sn, city, county,
           DATE_FORMAT(create_time, '%Y-%m-%d %H:%i:%s') AS create_time,
           water20cm, water40cm,
           CASE WHEN water20cm = 0 AND t20cm = 0 THEN 'device_fault'
                WHEN water20cm < 50 THEN 'heavy_drought'
                WHEN water20cm >= 150 THEN 'waterlogging'
                ELSE 'normal' END AS warning_level
    FROM fact_soil_moisture
    {where}
    ORDER BY create_time DESC
    LIMIT %s
    """
    params.append(int(limit))

    connection = self._connect()
    try:
        with connection.cursor() as cursor:
            cursor.execute(sql, tuple(params))
            rows = cursor.fetchall()
        return [dict(row) for row in rows]
    except Exception as exc:
        raise DatabaseQueryError(f"MySQL 查询预警记录失败：{exc}") from exc
    finally:
        connection.close()
```

```python
def count_warning_records_by_region(
    self,
    *,
    city: str | None = None,
    start_time: str | None = None,
    end_time: str | None = None,
) -> dict:
    """
    返回：{
        "total": 123,
        "by_warning_level": {"heavy_drought": 100, "waterlogging": 20, "device_fault": 3},
    }
    """
    clauses: list[str] = []
    params: list = []
    if city:
        clauses.append("city = %s")
        params.append(city)
    if start_time:
        clauses.append("create_time >= %s")
        params.append(start_time)
    if end_time:
        clauses.append("create_time <= %s")
        params.append(end_time)
    clauses.append("(water20cm < 50 OR water20cm >= 150 OR (water20cm = 0 AND t20cm = 0))")

    where = f"WHERE {' AND '.join(clauses)}" if clauses else ""
    sql_total = f"SELECT COUNT(*) AS cnt FROM fact_soil_moisture {where}"
    sql_by_level = f"""
    SELECT
        CASE WHEN water20cm = 0 AND t20cm = 0 THEN 'device_fault'
             WHEN water20cm < 50 THEN 'heavy_drought'
             WHEN water20cm >= 150 THEN 'waterlogging'
             ELSE 'normal' END AS warning_level,
        COUNT(*) AS cnt
    FROM fact_soil_moisture {where}
    GROUP BY warning_level
    """

    connection = self._connect()
    try:
        result: dict = {}
        with connection.cursor() as cursor:
            cursor.execute(sql_total, tuple(params))
            row = cursor.fetchone()
            result["total"] = int(row.get("cnt") or 0) if row else 0

            cursor.execute(sql_by_level, tuple(params))
            level_rows = cursor.fetchall()
            result["by_warning_level"] = {
                r["warning_level"]: int(r["cnt"] or 0)
                for r in level_rows
                if r.get("warning_level") != "normal"
            }
        return result
    except Exception as exc:
        raise DatabaseQueryError(f"MySQL 统计预警记录失败：{exc}") from exc
    finally:
        connection.close()
```

## 补丁：季节性预警暂停规则

> **此补丁已由原始计划作者直接落地**（`002_insert_data.sql` 和 `soil_repository.py` 已改），
> Codex 实现时只需在回答方法中调用 `is_warning_level_active()`，**不需要再改这两个文件**。

### 背景

`soil_warning_v1` 原始 JSON 缺少季节性规则：**每年 6/1–10/31 暂停涝渍预警，仅保留重旱与设备故障预警**。
已修复：`rule_definition_json` 新增 `seasonal_overrides` 数组，`soil_repository.py` 新增 `is_warning_level_active()` 静态方法。

### `is_warning_level_active()` 用法

```python
import json

rule_row = await self._soil_repo.warning_rule_row_async()
rule_def = json.loads(rule_row["rule_definition_json"]) if rule_row else {}

# 检查涝渍在当前日期是否有效（默认 check_date=today）
if not SoilRepository.is_warning_level_active("waterlogging", rule_def):
    # 当前处于夏季暂停期，涝渍预警无效
    ...
```

### 需要调用该方法的位置（Codex 实现时必须接入）

**1. `_reply_warning_rule_description()`**

在构建规则文本之前，获取 `rule_def`，然后：
- 对每条 rule，调用 `is_warning_level_active(rule["warning_level"], rule_def)` 判断当前是否激活
- 激活的规则正常展示，暂停的规则标注"（当前暂停）"
- 在回答末尾追加 `seasonal_overrides` 的描述文本（直接读 `override["description"]` 字段）

展示效果：
```
- 重旱预警：表层（20cm）相对含水量低于 50%
- 涝渍预警：表层（20cm）相对含水量高于或等于 150%（当前暂停，6月1日至10月31日不推送）
- 设备故障预警：表层含水量与温度同时为 0

特殊规则：每年6月1日至10月31日，暂停涝渍预警推送与判定，仅保留重旱预警与设备故障预警。
```

**2. `_reply_warning_list()` 和 `_reply_warning_count()`**

在调用 `query_warning_records_async` / `count_warning_records_by_region_async` 之前：

```python
rule_row = await self._soil_repo.warning_rule_row_async()
rule_def = json.loads(rule_row["rule_definition_json"]) if rule_row else {}

# 如果用户明确查涝渍，但当前涝渍已暂停，直接返回说明
if warning_type == "waterlogging" and not SoilRepository.is_warning_level_active("waterlogging", rule_def):
    return AgentTurnOutput(
        final_answer="当前处于夏季涝渍预警暂停期（6月1日至10月31日），该类型预警不推送与判定。",
        answer_type="guidance_answer",
        output_mode="normal",
        guidance_reason="boundary",
        ...
    )

# 如果是全类型查询（warning_type=None），从 SQL 条件中自动排除已暂停的等级
active_warning_sql_condition = _build_active_warning_condition(rule_def)
```

其中 `_build_active_warning_condition(rule_def)` 为内部辅助函数：

```python
def _build_active_warning_condition(rule_def: dict) -> str:
    import datetime
    today = datetime.date.today()
    conditions = []
    for level, sql_cond in [
        ("heavy_drought", "water20cm < 50"),
        ("waterlogging", "water20cm >= 150"),
        ("device_fault", "water20cm = 0 AND t20cm = 0"),
    ]:
        if SoilRepository.is_warning_level_active(level, rule_def, today):
            conditions.append(f"({sql_cond})")
    return " OR ".join(conditions) if conditions else "1=0"
```

把返回值拼入 SQL WHERE，替换原先硬编码的 `(water20cm < 50 OR water20cm >= 150 OR ...)` 字符串。

## 五、边界与注意事项

1. **`_classify_subject()` 参数扩展**：原始签名只有 `text`，现在增加 `has_city_entity` 和 `has_time_signal`。
   所有测试文件中直接调用 `_classify_subject` 的地方需要确认签名兼容（两个新参数都有默认值 `False`，不会破坏现有调用）。

2. **城市名归一化**：`soil_device_city_distribution` 和 `soil_device_county_distribution` 各自内部处理了"南京"→"南京市"的补全。
   如果后续需要在多个地方复用，可提取为 `_normalize_city_name(city: str) -> str` 私有静态方法。

3. **`fact_soil_moisture` 无 `soil_status` 列**：所有预警查询在 SQL WHERE 中直接写阈值，不走 Python `_evaluate_record_status()`。
   两者的阈值必须保持一致；P0 升级后统一改为从 `metric_rule` 表读取。

4. **`time_evidence_start`/`time_evidence_end` 传递**：
   在 `warning_record` 路由的 `_decision()` 调用里通过 `extra={"time_start": ..., "time_end": ...}` 传递，
   回答方法从 `(route_decision.extra or {}).get("time_start")` 取值。
   若 `AgentTurnDecision` 已有专门字段则直接使用。

5. **`_first()` 辅助函数**：`_reply_warning_list` 和 `_reply_warning_count` 中使用了 `_first(lst)` 工具函数，
   定义为 `lambda lst: lst[0] if lst else None`（或已有同名函数则直接引用）。

6. **`device_registry_county_detail` 路由中城市实体为空**：路由到此但 entities.city 为空时，
   返回 `guidance_reply(reason="clarification")`，不走全省查询（全省走 #5 的 distribution 路由）。

7. **`output_mode = "warning_mode"`**：预警记录回答使用此值。确认前端是否需要特殊渲染；
   若只作为元数据透传则不影响现有功能。

8. **SKILL.md 三处镜像**：路径如下，三个文件内容完全相同，必须同步更新：
   - `.claude/skills/soil-moisture-qa/SKILL.md`
   - `.agents/skills/soil-moisture-qa/SKILL.md`
   - `.codex/skills/soil-moisture-qa/SKILL.md`
