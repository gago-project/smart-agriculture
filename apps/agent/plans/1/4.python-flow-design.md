# 4. 墒情 Agent Python 工程方案

## 设计结论

当前工程坚持：

- 单 Agent
- LLM + Function Calling
- 5 节点受限 Flow
- 原始字段直通
- `create_time` 统一时间契约
- LLM 是决策者，代码是守门员

## 分层结构

```text
API
  -> Orchestrator (5 节点)
  -> Nodes (InputGuard, AgentLoop, DataFactCheck, AnswerVerify, FallbackGuard)
  -> Services (AgentLoopService, ToolExecutorService, InputGuardService, FactCheckService, AnswerVerifyService)
  -> Repositories (SoilRepository, SessionContextRepository)
  -> LLM Adapter (QwenClient + SOIL_TOOLS + SystemPrompt)
```

## 目录角色

- `apps/agent/app/api/`：HTTP 接口
- `apps/agent/app/flow/`：节点、路由表、编排器
- `apps/agent/app/services/`：业务逻辑、工具执行、Agent 循环
- `apps/agent/app/repositories/`：MySQL / Redis 访问
- `apps/agent/app/schemas/`：`FlowState`、枚举、DTO
- `apps/agent/app/llm/`：QwenClient、工具 Schema、System Prompt

## Orchestrator

编排器只负责三件事：

- 按固定顺序驱动节点
- 根据节点动作跳转
- 在终态前统一收口日志与异常

## AgentLoopService 内部循环

```text
load_history(session_id)
build messages = [system_prompt, ...history, user_message]
for i in range(MAX_TOOL_ITERATIONS=5):
    response = qwen.call_with_tools(messages, SOIL_TOOLS)
    if response.type == "text":
        save_message_turn(session_id, turn_id, ...)
        return AgentLoopResult(final_answer=response.content)
    if response.type == "tool_call":
        try:
            result = tool_executor.execute(tool_name, tool_args)
            append tool result to messages
        except ToolValidationError as e:
            append error to messages (LLM can adjust and retry)
return AgentLoopResult(final_answer=FALLBACK_MAX_ITER, is_fallback=True)
```

## FlowState 核心字段

| 字段 | 来源节点 | 说明 |
|------|----------|------|
| `request_id` / `session_id` / `turn_id` | 请求注入 | 标识 |
| `user_input` | 请求注入 | 原始用户输入 |
| `input_type` | InputGuardNode | 输入分类 |
| `answer_bundle.final_answer` | AgentLoopNode | 最终回答 |
| `query_result.records` | AgentLoopNode | 工具返回的记录 |
| `answer_type` | AgentLoopNode / DataFactCheckNode | 回答类型 |
| `conversation_closed` | InputGuardNode | 是否关闭话题 |
| `node_trace` | FlowRunner | 执行路径 |

## SessionContextRepository 接口

```python
async def load_history(session_id: str) -> list[dict]
async def save_message_turn(
    session_id: str, turn_id: int, *,
    user_message: str,
    assistant_message: str,
    tool_calls: list[dict],
    tool_results: list[dict],
) -> None
async def clear_context(session_id: str) -> None
```

Redis key: `session_msg:{session_id}`，最大 20 条消息，TTL 7200 秒。

## Repository 边界

### SoilRepository

- 只读 `fact_soil_moisture`
- 只暴露 `city`、`county`、`sn` 过滤
- 时间过滤统一是 `create_time`
- SQL 排序统一是 `ORDER BY create_time DESC`

### SessionContextRepository

- 保存 OpenAI messages 格式对话历史
- 清空话题时删除 Redis key

### QueryLogRepository

- 只记录真正执行过的查询
