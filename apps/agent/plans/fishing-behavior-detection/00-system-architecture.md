# 00. 水产养殖区域捕捞行为识别系统 — 系统架构与技术选型

## 1. 业务背景与核心问题

### 1.1 监管痛点

| 痛点 | 当前现状 | 目标 |
|------|---------|------|
| 禁捕区巡逻依赖人力 | 2-4 人轮班，成本高、覆盖盲区多 | 7×24h 无人值守自动预警 |
| 捕捞行为取证困难 | 人工拍照、证据链不完整 | 自动存证（视频+截图+轨迹） |
| 渔船夜间非法作业 | 夜间巡逻强度低 | 红外+可见光融合，全天候识别 |
| 人群聚集快速响应慢 | 发现延迟 10-30 分钟 | 聚集预警 ≤ 60s 送达执法人员 |
| 渔具类型无法区分 | 无法判断作业方式（拖网/围网） | 精确识别渔具类型，研判作业烈度 |

### 2.2 六层推理流水线（时序设计）

```
输入帧（1080P，25fps 原始流 → 抽帧 5-15fps）
    │
    ▼ ──── 预处理（< 5ms）
    │  is_nighttime → 红外融合增强
    │  运动区域检测（Background Subtraction）→ 感兴趣区域（ROI）裁剪
    │  Resize + Letterbox → 640×640 / 1280×1280（可配置）
    │
    ├─────────────── [L1] 人体检测（Object Detection）
    │  模型：YOLOv8m（TRT INT8）
    │  类别：person / person_with_gear（持渔具者）
    │  conf ≥ 0.45（水面低对比场景适当降阈值）
    │  NMS IoU ≤ 0.40（密集人群场景）
    │  输出：N 个人体 BBox + 置信度
    │
    ▼ ──── [L2] 人体跟踪（Multi-object Tracking）
    │  算法：ByteTrack（首选）/ DeepSORT（Re-ID 精度要求高时）
    │  输入：L1 检测框 + 历史轨迹
    │  输出：每个目标 track_id + 平滑后 BBox + 轨迹列表
    │  轨迹存储：Redis（滑动窗口 300s 历史）
    │  MOTA 目标：≥ 0.78，ID Switch ≤ 5次/h
    │
    ▼ ──── [L3] 区域聚集分析（Crowd Aggregation）
    │  输入：L2 跟踪结果（当前帧所有 track_id + 位置）
    │  Step A：密度估计（CSRNet 或 P2PNet）→ 人数统计（MAE ≤ 2）
    │  Step B：空间聚类（DBSCAN，eps=3m，min_samples=3）→ 聚集簇
    │  Step C：阈值判断（簇内人数 ≥ 5 且持续 ≥ 30s → 触发聚集预警）
    │  输出：聚集区域 BBox + 人数 + 持续时长 + 预警级别
    │
    ├─────────────── [L4+L5] 渔船 + 渔具检测（并行）
    │  ┌─── [L4] 渔船检测（Vessel Detection）
    │  │  模型：YOLOv8l（TRT FP16，大目标优先）
    │  │  类别：fishing_boat_small / fishing_boat_medium /
    │  │        fishing_boat_large / speedboat / unknown_vessel
    │  │  输入：全帧（渔船通常较大，不做 ROI 裁剪）
    │  │  输出：船只 BBox + 类别 + 置信度
    │  │
    │  └─── [L5] 渔具检测（Gear Detection）
    │     模型：YOLOv8m（TRT INT8）+ SAM 分割（可选，精细证据）
    │     类别：trawl_net（拖网）/ purse_seine（围网）/
    │           gill_net（刺网）/ fish_cage（养殖网箱，合法，排除）/
    │           fishing_rod（鱼竿）/ fish_trap（地笼）
    │     难点：渔具在水面的可见度低，需多尺度检测
    │     输出：渔具 BBox + 类别 + 与船只的关联关系
    │
    ▼ ──── [L6] 捕捞行为综合研判（Behavior Recognition）
    │  输入：L2 跟踪轨迹 + L3 聚集结果 + L4 船只 + L5 渔具
    │  研判维度（多维度融合规则引擎 + LSTM 时序模型）：
    │    • 船只静止时间（> 10min → 疑似作业）
    │    • 人员在船甲板聚集（≥ 3 人 + 持渔具 → 高疑）
    │    • 渔具入水动作（轨迹分析：物体从船上落水 → 确认捕捞）
    │    • 船只与渔具的空间关联（船周围 20m 内有渔具 → 关联）
    │    • 历史轨迹分析（往返规律性 → Trajectory Analysis）
    │  输出：
    │    SUSPICIOUS（疑似） → 保存证据片段，推送平台告警
    │    CONFIRMED（确认）  → 触发执法告警（微信+短信），存证
    │    NORMAL（正常）     → 仅日志记录
    │
    ▼ ──── 结构化输出 + 持久化
    {
      "event_id": "uuid-v4",
      "camera_id": "cam_shuiku_01",
      "timestamp": "2026-05-01T22:13:45.221Z",
      "behavior_type": "CONFIRMED",
      "confidence": 0.923,
      "evidence": {
        "person_count": 4,
        "persons_with_gear": 2,
        "crowd_clusters": [{"center":[x,y],"count":4,"duration_s":87}],
        "vessels": [{"class":"fishing_boat_small","conf":0.961,"bbox":[...]}],
        "gears": [{"class":"gill_net","conf":0.834,"bbox":[...]}],
        "trajectory_pattern": "stationary_with_periodic_movement"
      },
      "snapshot_url": "oss://...",
      "video_clip_url": "oss://...",    // 前后 30s 视频证据片段
      "inference_latency_ms": 412
    }
```

## 3. 技术选型决策矩阵

### 3.1 人体检测骨干网络

| 框架 | mAP（CrowdHuman） | FPS（Jetson Orin NX，INT8） | 小目标表现 | 推荐等级 |
|------|-----------------|---------------------------|-----------|---------|
| **YOLOv8m** | 88.3% | 32 FPS | ⭐⭐⭐⭐ | **主力（水面固定监控）** |
| YOLOv8l | 90.1% | 18 FPS | ⭐⭐⭐⭐⭐ | 云端精细化 |
| YOLOv8s | 84.7% | 55 FPS | ⭐⭐⭐ | 低算力边缘 |
| Faster R-CNN | 91.2% | 8 FPS | ⭐⭐⭐⭐⭐ | 离线精细分析 |
| DINO（Transformer）| 92.5% | 5 FPS | ⭐⭐⭐⭐⭐ | 研究验证 |

**决策**：实时检测用 YOLOv8m（边缘 INT8）；高精度离线取证用 YOLOv8l 或 Faster R-CNN。

### 3.2 人体跟踪算法

| 算法 | MOTA（MOT17） | ID Switch | 特点 | 推荐场景 |
|------|-------------|-----------|------|---------|
| **ByteTrack** | 80.3% | 低 | 低置信度框也参与跟踪，遮挡恢复好 | **主力**（密集人群）|
| DeepSORT | 75.2% | 中 | 引入 Re-ID 外观特征，跨遮挡效果好 | Re-ID 要求高时 |
| SORT | 65.1% | 高 | 纯卡尔曼+匈牙利，轻量 | 备用/资源紧张时 |
| OC-SORT | 78.9% | 低 | 改进版 SORT，处理非线性运动 | 快速移动目标 |
| StrongSORT | 79.5% | 低 | ByteTrack + ReID 联合 | 高精度跟踪 |

**决策**：默认 ByteTrack（边缘首选）；跨摄像头 Re-identification 场景升级为 StrongSORT。

### 3.3 人群密度估计

| 算法 | MAE（ShanghaiTech A） | 显存 | 实时性 | 推荐场景 |
|------|---------------------|------|-------|---------|
| **CSRNet** | 68.2 | 低 | ≥ 20 FPS | **主力**（密度图估计）|
| P2PNet | 52.7 | 中 | 15 FPS | 精确点计数（稀疏场景）|
| CrowdFormer | 45.3 | 高 | 8 FPS | 离线分析 |
| 计数：直接用 L2 跟踪结果 | ≈目标数量精确 | 极低 | 无额外消耗 | ≤ 30 人稀疏场景 |

**决策**：人数 ≤ 30 人（养殖区常见场景）直接用 L2 跟踪计数；> 30 人启用 CSRNet 密度估计。

### 3.4 渔船与渔具检测

| 任务 | 选型 | 理由 |
|------|------|------|
| 渔船检测（大目标） | YOLOv8l（TRT FP16） | 渔船目标大，精度优先 |
| 渔具检测（中小目标） | YOLOv8m + SAHI（切片推理） | 渔网等细长目标，SAHI 切片大幅提升小目标 mAP |
| 渔具精细分割 | SAM（Segment Anything Model，可选）| 生成精确 Mask，证据价值高 |
| 夜间渔船检测 | YOLOv8m + 红外融合通道 | 可见光+红外双通道输入 |

### 3.5 捕捞行为研判

```
研判模型层次设计（两级互补）：

Level 1：规则引擎（实时，< 10ms）
  - 快速研判，基于当前帧状态
  - 触发条件：船只停泊 + 人员聚集 + 渔具可见
  - 输出：SUSPICIOUS 级别（需 Level 2 确认）

Level 2：LSTM 时序模型（滞后 30s，< 50ms）
  - 输入：过去 30s 的跟踪轨迹序列 + 事件标签序列
  - 模型：双层 BiLSTM（输入维度 64，隐层 128）
  - 输出：CONFIRMED 概率
  - 优势：考虑时序上下文，大幅降低 FPR

两级联合研判：
  Level1=SUSPICIOUS AND Level2_prob ≥ 0.80 → CONFIRMED（触发执法告警）
  Level1=SUSPICIOUS AND Level2_prob < 0.80 → SUSPICIOUS（推送平台，不短信）
  Level1=NORMAL → NORMAL（日志记录）
```
