# 00. 水产运输车辆智能识别系统 — 系统架构与技术选型（企业级）

## 1. 项目背景与量化业务目标

### 1.1 核心问题

水产养殖主体在关键路口缺乏对运输车辆的有效监管，导致：
- 无法核实运输车辆是否为合规水产运输车（有水箱/罐体）
- 车牌信息采集依赖人工，效率低、漏记高
- 车辆归属主体无法快速核验（车身文字）
- 无人机航飞数据与固定监控数据无法联动分析

### 1.2 系统级 KPI 与验收标准

### 2.2 边缘节点 HA 故障切换

```
正常状态：
  主节点 (Master) ──── 处理所有推理任务
  备节点 (Standby) ─── 只做健康心跳，模型已加载待命

故障检测（心跳超时 10s）：
  Watchdog 进程检测主节点无响应
  → 备节点接管摄像头拉流
  → 告警推送运维人员
  → 切换完成 ≤ 30s（目标：≤ 15s）

恢复流程：
  主节点恢复 → 备节点继续工作（不立即切回，避免抖动）
  → 运维手动确认后，主节点重新接管
```

## 3. 四步算法流水线（精确版）

```
输入帧（BGR，≥ 1280×720，JPEG/PNG）
    │
    ▼ ──────────────── 预处理（< 5ms）
    │  is_nighttime? → CLAHE 增强（见 04 调优章节）
    │  Resize + Letterbox → 640×640，归一化 [0,1]
    │
    ▼ ──────────────── [Step 1] 目标检测（Object Detection）
    │  模型：YOLOv8m（TensorRT INT8，边缘）/ YOLOv8l（FP16，云端）
    │  任务：从全帧检测所有车辆（Classification 前置）
    │  输出：N 个 BBox（x1,y1,x2,y2,conf,class）
    │  阈值：conf ≥ 0.50，NMS IoU ≤ 0.45
    │  True Positive 条件：IoU ≥ 0.50（与 GT 框）
    │  指标监控：mAP50, mAP50-95, Precision, Recall
    │
    ├── 若无 truck/vehicle_aquatic 类型 BBox → 丢弃此帧
    │
    ▼ ──────────────── [Step 2] 水产车分类（Classification）
    │  输入：Step1 货车类 BBox 裁剪（扩边 10%，Resize 224×224）
    │  模型：YOLOv8m-cls（TRT INT8）
    │  特征：水箱/罐体结构、车顶充氧设备、车身文字特征
    │  输出：aquatic_prob ∈ [0,1]
    │  阈值：
    │    aquatic_prob ≥ 0.75 → 确认水产车（高置信）
    │    0.50 ≤ prob < 0.75 → 多模型投票（不确定区间）
    │    prob < 0.50 → 非水产车
    │  监控：False Positive Rate, False Negative Rate, F1-score
    │  时序滤波：5帧滑动窗口投票（减少单帧误判）
    │
    ▼ ──────────────── [Step 3 + 4 并行执行]
    │
    ├─────── [Step 3] 车牌识别（LPR：License Plate Recognition）
    │        3a 检测：YOLOv8n（TRT INT8）→ 车牌 BBox
    │        3b 增强：超分辨率（车牌宽 < 80px 时触发 ESRGAN 2×）
    │        3c 识别：CRNN（CNN+BiLSTM+CTC）→ 字符串 + confidence
    │        指标：Character Accuracy, Plate Accuracy
    │        置信度 < 0.80 → 加入 Active Learning 采样队列
    │
    └─────── [Step 4] 车身关键文字提取（OCR）
             输入：车辆 BBox 扩边 15%
             检测：DB++（文本行定位，Bounding Box + Polygon）
             识别：SVTR-LCNet → 文字串 + confidence
             后处理：关键词字典过滤（"渔""水产""养殖"等）
             输出：key_texts 列表 + has_aquatic_mark 布尔值
             指标：OCR Key Recall, False Positive（误识别率）
    │
    ▼ ──────────────── 结构化输出 + 持久化
    {
      "event_id": "uuid-v4",
      "frame_id": "cam01_20260501_143022_001",
      "camera_id": "cam_jiaokou_01",
      "timestamp": "2026-05-01T14:30:22.341Z",
      "vehicle_bbox": [245, 180, 890, 650],
      "detection_conf": 0.923,
      "is_aquatic_vehicle": true,
      "cls_confidence": 0.887,
      "cls_method": "single_model",          // or "ensemble"
      "license_plate": "粤B·12345",
      "lp_confidence": 0.971,
      "lp_bbox": [310, 580, 620, 640],
      "key_texts": [
        {"text": "渔运0023", "conf": 0.934, "is_key": true},
        {"text": "某某水产", "conf": 0.891, "is_key": true}
      ],
      "has_aquatic_mark": true,
      "snapshot_url": "oss://bucket/snapshots/...",
      "inference_latency_ms": 118,
      "pipeline_stage_latency_ms": {
        "preprocess": 3, "detection": 45, "classification": 28,
        "lpr": 22, "ocr": 35, "postprocess": 5
      }
    }
```

## 4. 技术选型决策矩阵

### 4.1 检测骨干网络选型

| 框架 | mAP50（COCO） | 推理速度（T4） | 参数量 | 边缘可部署 | 社区活跃度 | 推荐等级 |
|------|--------------|--------------|--------|-----------|-----------|---------|
| **YOLOv8m** | 50.2% | 234 FPS | 25.9M | ✅ TRT INT8 | ⭐⭐⭐⭐⭐ | **主力** |
| YOLOv8l | 52.9% | 126 FPS | 43.7M | ✅ TRT FP16 | ⭐⭐⭐⭐⭐ | 云端精细化 |
| YOLOv8n | 37.3% | 1040 FPS | 3.2M | ✅ INT8 | ⭐⭐⭐⭐⭐ | 车牌检测专用 |
| RT-DETR-l | 53.0% | 114 FPS | 32M | ⚠️ 复杂 | ⭐⭐⭐ | 研究验证 |
| YOLOv7 | 51.4% | 161 FPS | 36.9M | ✅ | ⭐⭐⭐ | 备选 |

**决策**：主力 YOLOv8m（边缘 INT8）+ YOLOv8l（云端 FP16），车牌检测用 YOLOv8n。

### 4.2 精度-速度权衡（边缘端 Jetson Orin NX）

```
模型量化精度对比（同 YOLOv8m，车辆检测任务）：

FP32：mAP50=0.891，延迟=310ms，显存=8.2GB  ❌ 超出延迟要求
FP16：mAP50=0.889，延迟=165ms，显存=4.1GB  ⚠️ 边界可用
INT8：mAP50=0.882，延迟=85ms， 显存=2.3GB  ✅ 推荐（精度损失<1%）

结论：生产边缘端使用 INT8，精度损失可接受，延迟/显存大幅改善
      INT8 校准数据集：从生产场景中采样 1000 张代表性图片
```

## 5. 模型压缩技术路线

生产环境中采用三阶段压缩降低推理成本：

```
原始训练模型（FP32，YOLOv8m）
        │
        ▼ 阶段1：剪枝（Pruning）
        │  去除冗余权重（L1/L2 结构化剪枝）
        │  目标：参数量减少 30%，mAP 损失 ≤ 0.5%
        │  工具：torch-pruning 库
        │
        ▼ 阶段2：知识蒸馏（Knowledge Distillation）
        │  Teacher 模型（YOLOv8l，精度高）→ 指导 Student（YOLOv8n）
        │  损失函数：L = α·L_task + β·L_distill（KL 散度）
        │  目标：Student 接近 Teacher 精度，推理速度提升 3×
        │
        ▼ 阶段3：量化（Quantization）
        │  训练后量化（PTQ）：INT8，TensorRT 校准
        │  量化感知训练（QAT，可选）：精度要求极高时使用
        │  目标：推理速度再提升 2×，显存减半
        │
        ▼ 压缩后模型（INT8，~YOLOv8n 大小）
           边缘端部署目标：延迟 ≤ 85ms，显存 ≤ 3GB
```
