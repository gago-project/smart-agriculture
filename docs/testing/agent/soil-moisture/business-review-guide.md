# 墒情 Agent 商务评审指南

本指南给商务、产品、运营同事使用，帮助大家在看测试样例或 Excel 结果时，把反馈集中在最有价值的地方。

## 1. 这轮评审主要看什么

请优先关注以下 4 件事：

- 问法是否像真实用户会说的话
- 回答是否能直接被业务理解和使用
- 模板类输出是否符合日常沟通习惯
- 澄清、无数据、超范围时的回复是否合理

## 2. 不需要重点关注什么

以下内容主要由研发和测试负责，不需要商务同事逐条确认：

- SQL 模板编号是否正确
- 内部路由、门禁、日志字段是否命中
- 代码实现方式、表结构、缓存细节

## 3. 建议按问题类型给反馈

建议把反馈尽量落到以下类别：

- 墒情概览：是不是像“汇总播报”
- 排名对比：排序口径是否符合业务理解
- 地区详情：地区名、时间、状态是否说清楚
- 设备详情：设备异常、最近数据是否说清楚
- 异常分析：重点关注对象是否准确、是否好理解
- 预警模板：正文是否像正式模板，解释是否多余或不够
- 指标解释 / 建议：是不是太技术化、是否对农户/大棚场景友好
- 澄清引导：问题问不清时，系统追问是否自然
- 无数据 / 兜底：没有数据时是否会误导
- 能力边界：超范围问题是否答得克制

## 4. 一条高质量反馈应该包含什么

建议尽量写清楚：

- 哪个 Case 或哪条问题
- 你觉得哪里不对
- 更希望系统怎么说
- 这个问题属于“表达问题”还是“能力问题”

示例：

- “这条预警模板太像技术说明，不像发给业务人员的话术，建议更口语一点。”
- “这条地区详情里先讲了很多背景，真正结论太靠后，建议先给结论再给解释。”
- “这条澄清太泛，最好直接给 3 个常见可选问法。”

## 5. 推荐配套材料

给商务同事发测时，建议同时提供：

- 一份简化后的 Excel 结果表，只保留问题分类、用户问法、系统回答、是否符合预期、备注
- 本文档，帮助大家知道要从什么角度给反馈
- 如有模板输出类问题，额外挑几条重点模板单独标注

## 6. 评审结论怎么用

商务评审反馈通常分成两类：

- 表达优化：回答顺序、措辞、模板口吻、澄清方式
- 能力缺口：类型没覆盖、口径不对、边界处理不合理

前者优先回写到问答模板与回答策略，后者优先回写到 `apps/agent/plans/` 的能力设计，再同步更新测试矩阵。
